## 7. Backend — Multi-Agent System

**New directory:** `backend/agents/`

### `data_service.py` — F1DataService
Wraps FastF1's Ergast interface with TTL caching (300s standings/schedule, 3600s results).

| Method | Description |
|---|---|
| `get_driver_standings(season)` | Championship standings |
| `get_constructor_standings(season)` | Constructor standings |
| `get_race_results(season, round)` | Race results |
| `get_qualifying_results(season, round)` | Qualifying grid + times |
| `get_race_schedule(season)` | Full season calendar |
| `get_drivers(season)` | All drivers (limit=100 to avoid pagination cutoff) |
| `get_constructors(season)` | All constructors |
| `get_next_race()` | Next upcoming race |
| `get_driver_season_results(driver_id, season)` | All results for one driver |
| `get_h2h_drivers(d1, d2, season)` | Driver H2H: points, wins, podiums, DNFs, qualifying/race tallies |
| `get_h2h_constructors(t1, t2, season)` | Constructor H2H |

### `base.py` — BaseAgent
Abstract base class with `_call_llm(system_prompt, messages)` calling `meta-llama/llama-3-3-70b-instruct` via watsonx SDK.

### `router.py` — AgentRouter
Keyword-based intent classifier → routes to `H2HAgent` or `BriefingAgent` (catch-all).

| Intent keyword examples | Routes to |
|---|---|
| "vs", "compare", "h2h", "versus" | H2HAgent |
| "standings", "qualifying", "explain", "what is" | BriefingAgent |
| (everything else) | BriefingAgent |

### `h2h.py` — H2HAgent
- Parses comparison queries: "X vs Y", "compare X and Y", "who is faster X or Y"
- Resolves driver/team names, codes, nicknames (Max → verstappen, Checo → perez, etc.)
- If only one entity provided + user has a favourite → compares against favourite
- Fetches full season H2H data and formats compact stats block for the LLM

### `briefing.py` — BriefingAgent
Sub-intent classifier handles:
- `driver_standings` / `constructor_standings`
- `qualifying` / `race_result` / `session_summary`
- `next_race` — upcoming race info
- `personal` — user's favourite driver/team results
- `terminology` — DRS, tyre deg, undercut, safety car, etc.
- `general` — open F1 Q&A

Fetches relevant F1 data, injects as context block before LLM call. Respects user's `fav_driver`/`fav_team` from JWT profile.

**Updated:** `backend/endpoints/chat.py`
- `POST /api/chat/watsonx` now lazy-initialises `AgentRouter` and routes through it
- Builds `user_context` from authenticated user's profile fields

## 8. Bug Fixes

| Bug | Fix |
|---|---|
| `passlib` / `bcrypt` 4.x incompatibility → 500 on signup | Replaced `CryptContext` with direct `bcrypt.hashpw` / `bcrypt.checkpw` in `services/auth.py` |
| Wrong watsonx model ID (`granite-3-3-8b-instruct` not in instance) | Switched to `meta-llama/llama-3-3-70b-instruct` |
| Ergast default pagination limit 30 — Verstappen (and others) missing | Added `limit=100` to `get_driver_info` and `get_constructor_info` calls |
| `driverCode` field is `NaN` float for test/practice-only drivers | Added `str(raw_code) != "nan"` guard in `_resolve_driver` |
| H2H formatter accessed `name`/`code`/`dnfs`/`race_h2h` fields missing from summary dict | Enriched summary with `_driver_meta()` and added DNF/race-h2h tracking in the loop |

## 9. Setup / Migration Notes

```bash
# Re-initialise DB (role column added to chatHistory)
cd backend && rm app.db
conda run -n pitwall-backend python -c \
  "import models; from db.session import engine; from db.base import Base; Base.metadata.create_all(engine)"

# Install new Python deps
conda run -n pitwall-backend pip install fastf1 ibm-watsonx-ai

# Add to backend/.env
WATSONX_API_KEY=your_key
WATSONX_PROJECT_ID=your_project_id
WATSONX_URL=https://us-south.ml.cloud.ibm.com   # default, optional
```
